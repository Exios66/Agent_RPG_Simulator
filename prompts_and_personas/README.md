# Prompts & personas library

Authoring pack at repo root: **character psychology, voice, methodology, and injectable prompts**
aligned with procedural archetypes in `agent_rpg.random_scenario` (`_ARCHETYPES`).

## Layout

- `CATALOG.yaml` — index of all personas by `slug` and canonical `archetype` string (regenerated by the tool below).
- `personas/<slug>/` — one folder per persona:
  - **`manifest.yaml`** — **source of truth** (motivations, lore, methodology, constraints, relationships, voice, prompts, integration hints).
  - `motivations.md`, `lore.md`, `methodology.md`, `relationships.md`, `voice.md`, `constraints.md`, `prompts.md` — regenerated from the manifest for easy reading and diffs.
  - `injection_block.md` — **single file to paste** into an agent system prompt or external tool (regenerated).
- `_template/manifest.example.yaml` — copy when adding a new persona folder.

## New persona

1. Copy `_template/manifest.example.yaml` to `personas/<your_slug>/manifest.yaml` and fill all sections.
2. Run `python prompts_and_personas/tools/build_library.py` to emit the `.md` files, `injection_block.md`, and refresh `CATALOG.yaml`.
3. Optionally hand-edit the `.md` files afterward; re-running the script **overwrites** them from the manifest.

## Wiring into Agent RPG

- **Quick paste:** append `injection_block.md` to `AgentConfig.system_prompt` (and clear `prompt_template_id` if you want raw text only), *or*
- **Jinja variables:** keep `prompt_template_id: default` and add keys to `prompt_variables` in YAML, then extend `src/agent_rpg/templates/agents/default.jinja2` to reference them (e.g. `{{ persona_core }}`).
- **Archetype string:** set `archetype:` in scenario YAML to match `manifest.yaml` → `archetype` so procedural/random scenarios line up with this library.

## Regeneration

```bash
python prompts_and_personas/tools/build_library.py
```
